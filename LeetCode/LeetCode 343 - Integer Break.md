# LeetCode 343 - Integer Break

## 想法

從簡單的例子開始，`2` 可以分解成 `1 + 1`，沒有其他分解方式了，因此 `2` 的最大乘積為 `1 * 1 = 1`；`3` 的最大乘積為 `2`；`4` 的最大乘積為 `4`

如果要分解 `6`，以下為分解方式

```text
1 * 1 * 1 * 1 * 1 * 1 = 1
2 * 1 * 1 * 1 * 1 = 2
2 * 2 * 1 * 1 = 4
2 * 2 * 2 = 8;
3 * 1 * 1 * 1 = 3
3 * 2 * 1 = 6
3 * 3 = 9
4 * 1 * 1 = 4
4 * 2 = 8
5 * 1 = 5
```

我們發現，`6` 的分解中，挑出 `4 + 2` 這組，如果我們知道 `4` 最大乘積為 `2 * 2`，則 `4 + 2` 的最大乘積必為 `2 * 2 * 2`，以此方式對 `6` 的每個分解組合檢查一次，即可得到 `6` 的最大乘積

此時遇到問題，由先前算法，我們算出 `3` 的最大乘積為 `2`，但用這種求解法，`3 + 3` 這組會得到最大乘積為 `2 * 2 = 4`，顯然不正確。實際上，因為題目規定數字必須分解成 2 個以上，因此會 `3` 不能單獨成為自己的最大乘積，但在子問題求解時，我們不用給予此限制，所以我們讓 `3` 可以做為自己的最大乘積，這樣在求解 `6` 的分解組合 `3 + 3` 時，就會得到正確答案 `3 * 3 = 9`

## 解法

以 `dp[n]` 陣列儲存數字 `i` 的最大乘積，對於 `i < n` 的子問題，數字 `i` 必定存在 `i` 本身作為最大乘積，因此計算 `i` 的最大乘積時，先將 `dp[i]` 設為 `i`，然後對於 `i` 剩下的每個組合 `i - j` 和 `j`，檢查 `dp[i - j] * j` 是否大於 `dp[i]`，如果是，則更新 `dp[i]` 的值

計算 `n` 的最大乘積時，由於 `n` 必須由兩個數字以上組成，因此 `n` 的最大乘積不包含 `n` 本身，讓 `dp[n]` 的初始值設為 0 開始計算

## 程式碼

```cpp
class Solution {
public:
    int integerBreak(int n) {
        vector<int> dp(n + 1, 0);
        for (int i = 1; i < n; i++) {
            dp[i] = i;
            for (int j = 1; j < i; j++) {
                if (dp[i] < dp[i - j] * j) {
                    dp[i] = dp[i - j] * j;
                }
            }
        }

        for (int i = 1; i < n; i++) {
            if (dp[n] < dp[n - i] * i) {
                dp[n] = dp[n - i] * i;
            }
        }

        return dp[n];
    }
};
```
